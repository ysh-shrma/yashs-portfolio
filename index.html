The best product intuition comes from building. AI lets me stay in that loop without context-switching to a different team's roadmap.

Beyond Building: Claude Code as a Daily PM Tool

The projects above are the visible output. Underneath them is a persistent PM operating system I've built inside Claude Code: a structured repo that handles spec writing, stakeholder simulation, data analysis, and documentation across 27+ project folders.

Persistent Context, Not Cold Starts

My Claude Code workspace isn't a chat window. It's a structured repository with ground truth context, enforced constraints, and compounding knowledge.

  • Every session starts grounded in product context. Claude reads the product context file, dealer economics model, and domain knowledge before any work begins. No hallucinated capabilities.
  • Every feature is anchored to dealer economics. A car sitting on the lot costs ~$50/day. If a feature doesn't reduce days-in-inventory or protect gross margin, it doesn't ship.
  • Enforced writing style. Plain language, active voice, no filler, no jargon. "Response time dropped from 3 days to 4 hours": not "significant improvement in key metrics."
  • Memory across sessions. Decisions, open items, who reviewed what: Claude picks up where we left off. Six months of PM work, continuously compounding.

Agent Performance Analytics

I built a full analytics pipeline to run weekly agent performance reviews: something that would normally require a dedicated analyst.

  • 7 Python scripts processing real call data: intent classification, outcome tiering, duration bucketing, transcript sampling
  • Weekly reports covering 450+ calls across 18 dealerships with tiered call funnels and intent-level resolve rates
  • Every finding tied back to business impact: which intent gaps cost the most in lost appointments, what prompt changes would move the needle

What Ships Through This System

Everything from feature specs to compliance research runs through the same Claude Code workflow:

  • Feature specs: CRM integration improvements, appointment booking nudge frameworks, outbound agent designs, SMS-first agent requirements: each persona-reviewed and anchored to dealer ROI
  • Conversation design: Complete flows across 4 customer journeys (lease-end, trade-in, service cross-sell, appointment confirmation) with 22+ test scenarios
  • Compliance research: TCPA, FCC, state-level DNC requirements for AI voice agents: disclosure, recording consent, opt-out tracking, PEWC for outbound
  • Evaluation frameworks: LLM-based agent eval system across 6 dimensions (information accuracy, tool usage, conversation quality, escalation handling, conversion) running on every call

How I Work with Claude Code

1

Start with the user problem, not the tech

I describe what the user needs to accomplish: "A dealer principal needs to know in 30 seconds if they're bleeding money on aged inventory." The product thinking drives the conversation; the code follows.

2

Iterate through conversation, like a pairing session

I treat Claude Code like a senior engineer sitting next to me. I review output, push back on UX decisions, ask for alternatives, and refine until the product feels right.

3

Validate with real users, not just stakeholders

The output isn't a mockup: it's a running application. I put the Spyne dashboard in front of BDC managers, dealer GMs, and industry veterans. They clicked through it, stress-tested the numbers, and gave feedback that reshaped the product.

4

Ship working software, not slide decks

Both projects are deployed and live. Working software creates conversations that wireframes can't.
